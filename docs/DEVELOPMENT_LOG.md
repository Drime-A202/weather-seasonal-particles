# 开发过程记录

## 第 1 天：概念与框架

- 确定项目方向：天气数据驱动的粒子可视化
- 技术选型：Streamlit + Canvas（轻量、快速迭代）
- 搭建基本结构：`app.py` + `requirements.txt`
- 集成 Open-Meteo API，实现北京天气数据获取

## 第 2 天：粒子系统初版

- 实现 Canvas 粒子动画循环
- 粒子数量：200，随机运动
- 配色方案：单色渐变
- 定义天气分类函数（晴/雨/雪/雷暴/雾）
- 每种天气设置不同的粒子形态：
  - 晴：圆形缓慢浮动
  - 雨：细长斜线快速下落
  - 雪：大粒子缓慢飘落
  - 雷暴：小粒子随机弹跳 + 闪电特效
  - 雾：超大粒子缓慢扩散
- 粒子密度与风速、湿度关联
- 设计节气映射表（24 节气 → 配色 + 粒子参数）
- 天气代码 → 节气名的映射规则
- 背景渐变与山影叠加
- 侧边栏城市切换
- 切换时粒子自动刷新
- HUD 显示实时数据
- 调整各节气配色，确保差异明显
- 优化粒子运动物理（速度阻尼、边界处理）
- 添加点击涟漪效果

## 第 3-4 天：文档与部署

- 编写 README.md
- 编写 AI 协作说明
- 编写开发日志
- 推送到 GitHub
- 部署到 Streamlit Cloud

## 技术决策记录

| 决策 | 原因 |
| :--- | :--- |
| 使用 Streamlit 而非 Flask | 无需前端路由，开发速度更快 |
| 使用 Canvas 而非 Three.js | 2D 粒子效果足够，无 3D 需求 |
| 粒子数量动态调整 | 兼顾性能与视觉效果 |
| 节气映射基于日期+天气 | 比纯天气匹配更丰富 |
